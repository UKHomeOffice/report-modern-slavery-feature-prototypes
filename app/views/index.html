Report modern slavery Feature prototypes

Save and exit prototype

This page explains how to carry out user testing with this prototype. The prototype does not actually do Save and Return. It attempts to fake the behaviour.

As the user researcher, you'll need to use an email service such as your Home Office account to send a fake email to mirror the behaviour of the actual service.

There are various scenarios of how save and retun will work. Each scenario explains what the user will need to do and what the user researcher will do for testing.

Scenario 1: The first application

The report modern slavery landing page

The user will be filling out the form for the first time to report modern slavery and will need to supply their work email and organisation as part of the login process, in order to obtain a link to start the application process for that report.

The user researcher should take a note of the user's email and send a copy of an email to the user with the link below

Fill Out first application

The user will go to their email and click on a link above. It's best if the user is given a report of the victim of modern slavery so that the user enters predefined answers. This will then get displayed later. The user will save and exit the application to be completed at a later date.

Scenario 2

Returning to Existing application via website

After an application has been saved, the user can return via the website to continue the application and change any answers they previously made. This will generate a new link that will be submitted to their email in order to continue the application

Returning to Existing application via email link

Via the email link the user will be able to return to their existing application and change any answers they previously made.

Scenario 3

Expired link (user has waited more than 30 minutes)

Via the email link the user will attempt to return to their existing application. However due to the exiparation of the link they will need to enter their email to request a new valid link.

Scenario 4

Application has been deleted after 28 days - user enters app via email link

Via the email link the user will attempt to return to their existing application. However since the application is deleted after 28 days they will be shown a message prompting them to start a new application.

Scenario 5

Application has been deleted after 28 days - user enters app via website

Via the webiste the user will attempt to return to their existing application by entering their email in order to receive a new link to continue.

Application has been deleted after 28 days - user clicks on the email link that was sent

However since the application is deleted after 28 days they will be shown a message prompting them to start a new application.

Scenario 6

Email link opens on a different device or browser so the cookie cannot be found

If the link from the email is opened on a different device or browser, the cookie relating to the report will not be visible. At this stage they are assigned another cookie, asked to confirm their email and then sent another link relating to the new cookie
